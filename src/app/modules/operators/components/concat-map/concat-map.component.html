O mergeMap é bem parecido com o switchMap. Ambos fazem uma troca de inscrição do observable, porém o mergeMap espera que o segundo observable seja completado para escutar o próximo e fazer a troca de inscrição.

Percebe-se que se clicar-mos duas vezes no botão seguidamente, ele executará duas vezes o fluxo, porém espera que a primeira junção acabe para juntar a segunda
